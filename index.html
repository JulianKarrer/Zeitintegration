Taylor Series: Implicit Assumptions

  • Taylor series is a power series $f(x)\approx \sum_{n=0}^\infty a_n x^n$
  • Unlike the Fourier Series, it fails at discontinuities
  • n-th order Taylor approximation assumes $C_n$ continuity

Plug in $\boldsymbol{x}({t+h})$ as a function $f$: \[\boldsymbol{x}({t+h})\approx \boldsymbol{x}(t) + h \frac{d\boldsymbol{x}(t)}{dt} + \frac{h^2}{2}\frac{d^2\boldsymbol{x}(t)}{dt^2} + \frac{h^3}{6}\frac{d^3\boldsymbol{x}(t)}{dt^3} + \mathcal{O}(h^4)\] or in other notation: \[\boldsymbol{x}^{t+h}\approx \boldsymbol{x}^t + h \boldsymbol{v}^t + \frac{h^2}{2}\boldsymbol{a}^t + \frac{h^3}{6}\frac{d^3\boldsymbol{x}(t)}{dt^3} + \mathcal{O}(h^4)\] $\Longrightarrow$ more information/points → higher order derivatives → higher accuracy

The simplest case: Euler methods

First order approximations:

Making Explicit Euler more accurate: Verlet

  • higher order approxiamtion = more accuracy
  • only 2nd derivative is known at $\boldsymbol{x}^{t}$
  • idea: consider previous points $\boldsymbol{x}^{t-h}, \boldsymbol{x}^{t-2h}, \dots$
\[\begin{align*} \boldsymbol{x}^{t+h}&\approx \boldsymbol{x}^t &+ h \boldsymbol{v}^t &+ \frac{h^2}{2}\boldsymbol{a}^t &+ \frac{h^3}{6}\frac{d^3\boldsymbol{x}(t)}{dt^3} &+ \mathcal{O}(h^4)\\ \boldsymbol{x}^{t-h}&\approx \boldsymbol{x}^t &- h \boldsymbol{v}^t &+ \frac{h^2}{2}\boldsymbol{a}^t &- \frac{h^3}{6}\frac{d^3\boldsymbol{x}(t)}{dt^3} &+ \mathcal{O}(h^4)\\ \end{align*} \]

The Landscape of Integration Methods

The simplest case: Euler methods

First order approximations:
\[\boldsymbol{x}^{t+h}\approx \boldsymbol{x}^t + h \boldsymbol{v}^t + \mathcal{O}(h^2)\] Explicit Euler, Forward Euler, Forward Difference
\[\boldsymbol{x}^{t+h}\approx \boldsymbol{x}^t + h\boldsymbol{v}^{t+h} + \mathcal{O}(h^2)\] Implicit Euler, Backward Euler, Backward Difference

the problem with implicit methods

  • if $\boldsymbol{v}$ is only a function in time, things are not interacting
  • usually, $\boldsymbol{v}$ and $\boldsymbol{a}$ are functions of position $\boldsymbol{x}(t)$
  • $\boldsymbol{x}({t+h})$ on both sides:
    \[\color{red}\boldsymbol{x}({t+h})\color{white} \approx \boldsymbol{x}(t) + h\boldsymbol{v}(\color{red}\boldsymbol{x}({t+h})\color{white})\]
  • implicit methods generally yield a non-linear equation we need to solve

Implicit Methods - solving the equation

\[\boldsymbol{x}^{t+h} = \boldsymbol{x}^t + h\boldsymbol{v}(\boldsymbol{x}^{t+h})\]
$\Longrightarrow$ Newton-Rhapson methods: \[0 = \boldsymbol{x}^t + h\boldsymbol{v}(\boldsymbol{x}^{t+h}) - \boldsymbol{x}^{t+h}\] $\Longrightarrow$ Linearization with Jacobian: \[\boldsymbol{x}^{t+h} = \boldsymbol{x}(t) + h\left( \boldsymbol{v}^t + \boldsymbol{J}^t h \boldsymbol{x}^{t+h} \right)\]
  • more work: solve an equation for each timestep
  • more error: linearization and solver inaccuracy
  • BUT: they can be unconditionally stable and handle stiff problems

The core problem - stiffness and stability



Definition « If a numerical method [...] is forced to use in a certain interval of integration a step length which is excessively small in relation to the smoothness of the exact solution in that interval »

  • stiff problem = timestep is limited by stability, not accuracy
  • → low variation and high smoothness, but small timestep required
  • frequently a problem for decaying functions: $e^{kx}$ for $k<0$

the archetypical stiff problem: A-stability


$v^t = -kx^t$
$x^t = e^{-kt}$
  • smooth and has low variation
  • still, instabilities occur → stiff problem!
  • a method is A-stable if it can deal with this problem: \[x^t \rightarrow 0 \text{ as } t \rightarrow \infty \]
